# Skill: image-optimization

> Local skill stub. Image pipeline.

## Strategy

- Source images live under `public/images/<category>/` and are optimized at build via `sharp`.
- Output formats: AVIF (best), WebP (good), JPG/PNG (fallback).
- `<picture>` element with `<source type="image/avif">`, `<source type="image/webp">`, `<img>` fallback.
- `loading="lazy"` everywhere except hero/LCP candidate.
- `decoding="async"`.
- `fetchpriority="high"` only on hero.
- Explicit `width` / `height` attributes to prevent CLS.
- `alt` text always meaningful (German), never empty unless decorative.

## Placeholder system

When real photo missing, render an SVG placeholder generated by
`src/lib/placeholder.ts` — markenkonform (gold icon on charcoal gradient).

The `<picture>` tag references the (probably missing) photo as `src` with an
`onerror` handler that swaps to the SVG. Once a real photo lands in
`public/images/meals/<slug>.jpg`, the placeholder disappears automatically.

## Logo pipeline (build-time)

`scripts/build-images.mjs` runs as `prebuild`:

1. Take `public/brand/logo.png` (transparent).
2. Emit: `logo.webp`, `logo.avif`, PWA icons (192, 512, maskable 192, maskable 512),
   favicons (16, 32, ico), `apple-touch-icon.png` (180×180 on `#0a0a0a`),
   `og-image.jpg` (1200×630 logo centered on `#0a0a0a`).
